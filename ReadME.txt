
1.实现了udp和tcp消息的交互。tcp采用nio技术实现，同时包含了消息组包。

2. 采用多线程技术实现消息的发送，接收和处理，极大提高了程序的效率。

3. MainActivity中有消息收发的Demo.修改ChannelProxy的参数即可分别测试Udp和Tcp消息的收发。

4. 消息的头十个字节分别为：
			头两个字节为消息长度（用来冗余消息的完整性，同时在tcp中用来组包）
			八个字节为消息时间戳（用来匹配请求和响应）
   消息结构为：2字节（长度） +８字节（发送消息的时间戳）　＋ n字节（实际的业务数据）

5. 如果需要处理推送的消息，需要注册推送消息的回调接口。推送消息的时间戳为-1.
